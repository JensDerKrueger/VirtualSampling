# Scripting Language and Automation

Raycaster includes a lightweight command interpreter for reproducible
measurements, batch screenshot generation, and scripted sweeps over datasets and
rendering modes.

Example scripts are included in:

- `Raycaster/Scripts/quality.gsc`
- `Raycaster/Scripts/performance.gsc`
- `Raycaster/Scripts/scaling.gsc`

## Running a script

The native application accepts a script via `--script`:

```bash
cd Raycaster/build
./Raycaster --script quality.gsc
```

The build copies the example scripts into `Raycaster/build/`, so the simple form
above works after a normal `make` or `make release`. You can also pass any other
relative or absolute script path.

## Language overview

A script is a plain text file processed line by line.

- Commands are whitespace-separated tokens.
- Blank lines act as separators between execution batches.
- `#` starts a comment.
- Command names are case-sensitive.
- There is no quoting mechanism for strings with spaces.

This design keeps scripts easy to parse and suitable for incremental execution
inside a render loop.

## Variables

Variables are stored as strings and substituted textually before a command parses
its arguments.

### Set and unset variables

```text
set n 3
set mode 2
set filename screenshot.png
unset mode
```

### Variable references

Use either `$name` or `${name}`:

```text
set n 3
repeat $n as $i
  setmethod $i
endrepeat
```

If a referenced variable does not exist, the interpreter reports an argument
error.

### Integer expressions in `set`

`set` supports a simple integer expression syntax using `+`, `-`, `*`, and `/`
with normal precedence.

```text
set i 0
set i $i + 1
set samples $i + 2 * 3
```

Parentheses are not currently supported.

## Control flow

### Delay with `noop`

```text
noop 200
```

This pauses further script execution for the specified duration in milliseconds.

### Repeat loops

```text
repeat 5
  screenshot
endrepeat
```

```text
repeat 5 as $i
  setmethod $i
  screenshot
endrepeat
```

The loop count must evaluate to a non-negative integer.

### Conditionals

Truthy form:

```text
if $enabled
  log enabled
endif
```

Comparison form:

```text
if $mode == 2
  log mode-two
else
  log other-mode
endif
```

Supported comparison operators:

```text
== != < <= > >=
```

If both sides parse as integers, numeric comparison is used. Otherwise `==` and
`!=` compare strings.

## Core command reference

These commands are registered by the shared `GLApp` base class.

### Reset and interaction

- `reset`
  Reset application state to a baseline.
- `setinteraction <enabled>`
  Enable or disable interactive controls while a script is running.

### Window and rendering

- `setbackground <r> <g> <b> <a>`
  Set the background color.
- `resize <width> <height>`
  Resize the render surface.

### Screenshots

- `screenshot`
  Save a screenshot with an autogenerated name in the current output directory.
- `screenshot <filename>`
  Save a screenshot with an explicit filename.

### Logging and FPS

- `setfpswindow <seconds>`
  Set the FPS accumulation interval.
- `clearlog`
  Remove the current log file if one is configured.
- `logfile <filename>`
  Set the log filename.
- `logtime`
  Append the current wall-clock time to the log.
- `logfps`
  Record the measured FPS for the current accumulation interval.
- `logGLInfo <includeExtensions>`
  Write OpenGL information to the log.
- `log <message...>`
  Append an arbitrary message to the log.

### Output and termination

- `setdir <directory>`
  Set the output directory for screenshots and logs. The directory is created if
  needed.
- `quit`
  Close the application.

## Raycaster-specific commands

These commands are registered by `Raycaster` itself.

### Dataset, method, and sampling

- `setvolume <index>`
  Select the active dataset.
- `setmethod <index>`
  Select the render mode.
- `setrate <rate>`
  Set the sampling rate.
- `setsubdiv <count>`
  Set the number of virtual samples.
- `constantSampleCount <enabled>`
  Toggle constant total sample count behavior.
- `setLevel <level>`
  Select the mip or downsampling level.

### Transfer function and filtering

- `settfparams <gaussian> <start> <width>`
  Configure the built-in Gaussian or Smoothstep transfer function.
- `settffile <filename>`
  Load a transfer function from file.
- `settfcode <encoded>`
  Load a transfer function from an encoded string.
- `setalphathreshold <threshold>`
  Set the alpha threshold used for early ray termination.
- `setusennfilter <enabled>`
  Toggle nearest-neighbor filtering.

### Camera and transform

- `setuseortho <enabled>`
  Toggle orthographic projection.
- `resetrotation`
  Reset the rotation matrix.
- `setrotation <16 floats>`
  Set the full rotation matrix explicitly.
- `addrotationx <degrees>`
  Rotate around the x axis.
- `addrotationy <degrees>`
  Rotate around the y axis.
- `addrotationz <degrees>`
  Rotate around the z axis.
- `settranslation <x> <y> <z>`
  Set the translation vector.
- `settransformparams <encoded>`
  Restore a transform from the URL-style encoded representation.

### Prefiltering

- `prefilter`
  Recompute all prefiltered volumes.

## Minimal examples

### Save screenshots for several methods

```text
setdir results/screens
setinteraction false
resize 512 512
setvolume 6
settfparams false 0.097 0.156

repeat 4 as $i
  setmethod $i
  screenshot method-$i.png
endrepeat
```

### Log performance

```text
setdir results/performance
logfile fps.log
clearlog
setfpswindow 5
setvolume 6
setmethod 13
logfps
quit
```

## Practical examples in this repository

The bundled scripts are good starting points:

- `quality.gsc` focuses on screenshot generation across many volumes and methods.
- `performance.gsc` performs repeated FPS logging sweeps.
- `scaling.gsc` varies render resolution and optionally captures images.

They also show a useful pattern: store reproducible camera states and transfer
functions directly in the script, often copied from a shared web URL.
